# Flink Manager Task List（基于 modification-suggestions.md）

## Phase 1（短期：马上可做，轻量高收益）

### P1-1 总览页增加 KPI 卡片
- [ ] 在 `index.html` 增加顶部 KPI 区域
- [ ] 指标包含：集群总数、在线集群数、Job 总数、RUNNING 数、FAILED 数、启用告警数
- [ ] 使用现有 localStorage 数据源（`flink_clusters_v1` / `flink_jobs_v1` / `flink_alerts_v1`）
- [ ] 空数据场景展示为 0，不报错

**验收标准**
- [ ] 打开首页即可看到 KPI 卡片
- [ ] 新增/删除集群、Job、告警后，返回首页数据同步变化

---

### P1-2 Job 页面增加参数详情弹窗
- [ ] 在 `jobs.html` 增加“查看参数”按钮
- [ ] 点击后弹窗展示完整 `params` JSON（格式化）
- [ ] 若无参数则展示“无参数信息”
- [ ] 弹窗支持关闭（右上角按钮 + 点击遮罩关闭）

**验收标准**
- [ ] 可查看解析后的完整参数
- [ ] 不影响现有新增/删除/解析并添加流程

---

### P1-3 Job 页面增加筛选能力
- [ ] 增加筛选项：集群、状态、关键字（匹配 jobname / database / bu / remark）
- [ ] 支持组合筛选
- [ ] 增加“重置筛选”按钮
- [ ] 筛选在前端本地完成，不改存储结构

**验收标准**
- [ ] 筛选结果正确
- [ ] 重置后恢复全量列表

---

### P1-4 告警页增加级别与规则条件
- [ ] 在 `alerts.html` 表单增加 `severity`（P1/P2/P3）
- [ ] 增加 `rule` 字段（文本或下拉 + 阈值输入）
- [ ] 列表展示级别与规则摘要
- [ ] 兼容旧告警数据（没有新字段时给默认值）

**验收标准**
- [ ] 新增告警可配置级别与规则
- [ ] 页面刷新后数据持久化正常
- [ ] 旧数据不会导致渲染异常

---

## Phase 2（中期：接入实时状态）

### P2-1 抽象数据访问层（为 API 接入做准备）
- [ ] 新增统一数据访问模块（本地模式/接口模式）
- [ ] 保持页面调用方式一致，减少后续改动面
- [ ] 默认先走 localStorage，保留切换开关

**验收标准**
- [ ] 页面在本地模式功能不退化
- [ ] 能通过配置切换数据来源

---

### P2-2 接入 Flink REST API（Job 运行态）
- [ ] 获取 Job 实时状态（RUNNING/FAILED/CANCELED 等）
- [ ] 获取基础运行信息（启动时间、运行时长、checkpoint 时间）
- [ ] 将实时态合并到 Job 列表展示
- [ ] 接口失败时优雅降级（展示“未知”）

**验收标准**
- [ ] Job 状态可实时刷新
- [ ] 接口异常不影响页面基本可用

---

### P2-3 集群连通性自动刷新
- [ ] 对已配置集群做定时可达性检查
- [ ] 展示在线/离线/未知 + 最近检测时间
- [ ] 支持手动“立即检测”

**验收标准**
- [ ] 状态可自动更新
- [ ] 手动检测可立即反馈结果

---

### P2-4 告警触发记录（简版）
- [ ] 新增告警触发记录数据结构（时间、告警名、级别、对象、状态）
- [ ] 在 `alerts.html` 增加“最近触发记录”区域
- [ ] 仅保留最近 N 条（例如 100 条）避免页面变重

**验收标准**
- [ ] 可查看近期告警触发历史
- [ ] 页面加载与交互保持流畅

---

## Phase 3（后期：增强效率与可视化）

### P3-1 批量运维动作（可选）
- [ ] Job 列表支持多选
- [ ] 支持批量停止/批量启动
- [ ] 批量操作前二次确认
- [ ] 成功/失败结果可见

**验收标准**
- [ ] 批量操作可执行且反馈清晰
- [ ] 误操作风险可控（确认弹窗）

---

### P3-2 趋势图与容量预警
- [ ] 总览页增加趋势图（失败次数、告警次数、在线率）
- [ ] 增加简单容量预警规则（如失败率超过阈值）
- [ ] 支持时间范围切换（24h/7d）

**验收标准**
- [ ] 趋势展示清晰
- [ ] 阈值触发时有可见提示

---

### P3-3 体验优化与稳定性收敛
- [ ] 统一页面样式与交互组件（按钮、弹窗、标签）
- [ ] 增加统一空态、加载态、错误态
- [ ] 补充关键页面自测清单与回归清单

**验收标准**
- [ ] 页面风格一致
- [ ] 常见异常路径可控
- [ ] 回归测试可重复执行

---
